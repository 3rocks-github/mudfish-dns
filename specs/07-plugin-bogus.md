# 플러그인: Bogus IP / Bogus NXDOMAIN

이 문서는 미꾸라지 DNS의 `bogus_ip` 및 `bogus_nxdomain` 플러그인에 대한 명세입니다.
이 플러그인들은 특정 ISP나 검열 기관(예: 중국의 GFW)에 의해 발생하는 DNS 오염(DNS Pollution)을 우회하기 위해 설계되었습니다.

## 1. 개요

DNS 오염 환경에서는 사용자가 쿼리한 도메인에 대해 의도적으로 조작된(가짜) IP 주소나 NXDOMAIN(존재하지 않는 도메인) 응답이 정상적인 응답보다 먼저 클라이언트에게 도달할 수 있습니다.
이 플러그인은 업스트림 서버로부터 돌아온 응답 패킷의 데이터를 검사하여, 알려진 가짜 IP 주소가 포함되어 있을 경우 해당 응답을 폐기(drop)하거나 `NXDOMAIN`으로 변조하여 실제 올바른 응답을 기다릴 수 있도록 합니다.

## 2. 문법 (Syntax)

### 2.1. `bogus_ip`

```text
bogus_ip <IP_ADDRESS...>
```

* **IP_ADDRESS**: DNS 오염에 사용되는 것으로 알려진 가짜 IPv4 또는 IPv6 주소 (또는 CIDR 서브넷) 목록입니다. 공백으로 구분하여 여러 개를 나열할 수 있습니다.
* **동작 방식**: 업스트림 서버로부터 받은 DNS 응답(A 또는 AAAA 레코드)의 IP 주소가 이 목록에 포함되어 있다면, 해당 응답 패킷을 무시(drop)합니다. 클라이언트는 이 가짜 응답을 받지 않고 진짜(느린) 응답이 오기를 계속 기다리게 됩니다.

### 2.2. `bogus_nxdomain`

```text
bogus_nxdomain <IP_ADDRESS...>
```

* **IP_ADDRESS**: 특정 환경에서 존재하지 않는 도메인을 쿼리했을 때, ISP가 광고나 안내 페이지로 강제 리다이렉트하기 위해 반환하는 가짜 IP 주소 목록입니다.
* **동작 방식**: 업스트림 서버로부터 받은 DNS 응답의 IP 주소가 이 목록에 포함되어 있다면, 해당 응답을 정상 IP 응답이 아닌 `NXDOMAIN` (RCODE 3) 에러 응답으로 변조하여 클라이언트에게 반환합니다.

## 3. 예제 (Examples)

### 3.1. DNS 오염 방지 (Bogus IP)

알려진 가짜 IP가 응답으로 오면 이를 폐기하여 실제 올바른 IP 응답을 기다립니다.

```text
. {
    forward . 8.8.8.8
    bogus_ip 127.0.0.1 243.185.187.30 243.185.187.39
}
```

### 3.2. ISP 광고 리다이렉션 방지 (Bogus NXDOMAIN)

존재하지 않는 도메인 쿼리 시 특정 ISP가 `10.10.10.10` 이라는 자사 광고 서버 IP를 반환하는 경우, 이를 감지하여 원래 기대했던 `NXDOMAIN` 오류로 처리합니다.

```text
. {
    forward . 8.8.8.8
    bogus_nxdomain 10.10.10.10
}
```
