# 플러그인: ACL (Access Control List)

이 문서는 미꾸라지 DNS의 `acl` 플러그인에 대한 명세입니다.
이 플러그인은 도메인 기반의 화이트리스트/블랙리스트 기능이 아니며, **[CoreDNS의 acl 플러그인](https://coredns.io/plugins/acl/)**과 동일한 목적과 문법을 가지도록 설계되었습니다. 주로 **클라이언트의 IP 주소(서브넷)** 및 **DNS 쿼리 타입**을 기반으로 접근을 제어(허용, 차단, 무시)하는 데 사용됩니다.

## 1. 개요

`acl` 플러그인은 특정 도메인 존(Zone)으로 들어오는 쿼리에 대해 클라이언트의 네트워크 대역이나 쿼리의 레코드 타입(A, AAAA, MX, ANY 등)을 검사합니다.
지정된 규칙들은 위에서부터 아래로 순차적으로 평가되며, 매칭되는 첫 번째 규칙의 동작(Action)이 즉시 적용됩니다.

## 2. 문법 (Syntax)

```text
acl [ZONES...] {
    <ACTION> <FILTER_TYPE> <VALUES...>
}
```

* **ZONES**: 이 ACL 규칙이 적용될 도메인 존입니다. 생략하면 상위 설정 블록에 정의된 존에 기본으로 적용됩니다.
* **ACTION**: 조건 매칭 시 수행할 동작을 정의합니다.
  * `allow`: 쿼리를 허용하고 다음 플러그인으로 처리를 진행합니다.
  * `block`: 쿼리를 차단하고 `REFUSED` (RCODE 5) 응답을 반환합니다.
  * `drop`: 쿼리에 응답하지 않고 즉시 패킷을 버립니다 (클라이언트 측에서 Timeout 발생).
* **FILTER_TYPE**: 검사할 쿼리의 속성입니다.
  * `net`: 클라이언트의 IP 주소 또는 CIDR 서브넷 (예: `192.168.1.0/24`, `10.0.0.5/32`, `2001:db8::/32`).
  * `type`: DNS 쿼리 타입 (예: `A`, `AAAA`, `ANY`, `MX`, `TXT`).
* **VALUES**: 필터링 대상이 되는 실제 네트워크 대역이나 쿼리 타입의 목록입니다. 공백으로 구분하여 여러 개를 나열할 수 있습니다.

## 3. 평가 규칙 (Evaluation)

1. 규칙은 블록 내에 나열된 순서대로 **하향식(Top-Down)**으로 평가됩니다.
2. 쿼리가 규칙의 조건과 일치하면 즉시 해당 `ACTION`이 실행되며 하위 규칙은 평가하지 않습니다.
3. 어떤 규칙과도 일치하지 않는 쿼리는 **기본적으로 허용(allow)**되어 다음 플러그인 체인으로 안전하게 넘어갑니다.

## 4. 예제 (Examples)

### 4.1. 특정 IP 대역 차단
내부망인 `10.0.0.0/8` 대역에서 오는 모든 쿼리에 대해 `REFUSED`를 반환하여 차단합니다.

```text
. {
    acl {
        block net 10.0.0.0/8
    }
}
```

### 4.2. 특정 쿼리 타입 무시 (Drop)
네트워크 대역폭을 소모하는 `ANY` 타입의 쿼리가 들어올 경우, 응답을 보내지 않고 무시합니다 (DDoS 증폭 공격 방어 등에 유용).

```text
. {
    acl {
        drop type ANY
    }
}
```

### 4.3. 화이트리스트 방식 네트워크 제어
로컬 호스트(`127.0.0.1`)와 특정 사내망(`192.168.1.0/24`)에서 오는 쿼리만 허용하고, 나머지 모든 IPv4 및 IPv6 대역의 접근을 차단합니다.

```text
. {
    acl {
        allow net 127.0.0.1/32
        allow net 192.168.1.0/24
        block net 0.0.0.0/0
        block net ::/0
    }
}
```
