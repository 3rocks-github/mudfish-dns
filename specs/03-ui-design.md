# 미꾸라지 DNS UI 명세 (UI Specification)

이 문서는 미꾸라지 DNS의 사용자 인터페이스(UI) 구성 요소 및 사용자 경험(UX) 요구 사항을
정의합니다. `crates/ui` 프로세스는 이 명세를 바탕으로 구현되어야 합니다.

## 1. 개요

미꾸라지 DNS의 UI는 사용자가 복잡한 DNS 설정과 네트워크 상태를 직관적으로 이해하고 제어할
수 있도록 돕는 핵심 구성 요소입니다. 백그라운드에서 동작하는 Core Process 및 Service
Process와 IPC를 통해 안전하게 통신하여 상태를 동기화하고 명령을 내립니다.

## 2. 디자인 원칙 (Design Principles)

- **직관성 (Intuitive):** 사용자가 네트워크나 DNS 프로토콜에 대한 깊은 지식이 없어도 쉽게
  보호 기능을 켜고 끌 수 있어야 합니다.
- **가시성 (Visibility):** 현재 DNS의 연결 상태, 사용 중인 업스트림 서버, 실시간 차단 통계
  등을 한눈에 파악할 수 있는 대시보드를 제공해야 합니다.
- **접근성 (Accessibility):** 시스템 트레이(상태 표시줄)에 상주하여 언제든 빠르게 접근할
  수 있어야 하며, 다크 모드/라이트 모드를 지원하여 사용자 환경에 맞게 제공되어야 합니다.

## 3. 주요 화면 구성 (Screen Layout)

앱은 크게 사이드바(또는 상단 탭)를 통한 메뉴 내비게이션과 우측의 메인 콘텐츠 영역으로 구성됩니다.

### 3.1. 메인 대시보드 (Main Dashboard)

앱 실행 시 가장 먼저 표시되는 화면으로, 미꾸라지 DNS의 전반적인 상태와 요약 정보를 보여줍니다.

* **상태 및 제어 (Status & Control):**
  * **Start / Stop 버튼:** 크고 명확한 토글 버튼으로 전체 DNS 보호 서비스의 켜기/끄기
    상태를 제어합니다.
  * **현재 상태 표시:** "보호 중 (Protected)", "연결 해제됨 (Disconnected)",
    "연결 중 (Connecting...)" 등의 텍스트와 시각적 색상(초록/회색/노랑)으로 상태를
    표시합니다.
  * **현재 업스트림 서버:** 현재 활성화된 업스트림 DNS 서버의 정보(IP 주소, DoH/DoT 등
    사용 중인 프로토콜)와 실시간 응답 속도(Latency)를 간략히 표시합니다.

* **실시간 통계 요약 (Real-time Statistics):**
  * **총 쿼리 수:** 서비스 시작 후 처리된 전체 DNS 쿼리의 수.
  * **차단된 쿼리 수:** 플러그인(광고/트래커/유해 도메인)에 의해 차단된 쿼리 수 및 전체
    대비 비율.
  * 트래픽 추이를 보여주는 간단한 시각적 차트(선 그래프 또는 파이 차트).

### 3.2. 쿼리 로그 (Query Log)

기기에서 발생하는 DNS 쿼리 내역을 실시간으로 확인하는 화면입니다.

* **로그 목록 표시:** 시간, 요청한 도메인 이름, 쿼리 타입(A, AAAA 등), 응답 상태(허용됨,
  차단됨, 캐시 적중 등), 사용된 업스트림 서버 정보를 표(Table) 형태로 제공합니다.
* **필터링 및 검색:**
  * 특정 도메인 키워드 검색.
  * '차단된 쿼리만 보기', '캐시된 쿼리만 보기' 등의 필터 토글.
* **데이터 관리:**
  * 로그 기록 기능 On/Off 토글 스위치 (개인정보 보호 목적).
  * 화면에 표시된 로그 지우기 (Clear Logs) 버튼.

### 3.3. 설정 (Settings)

미꾸라지 DNS의 세부 동작 규칙을 구성하는 화면입니다. 설정은 논리적인 그룹으로 나뉘어 표시됩니다.

* **업스트림 DNS (Upstream DNS):**
  * 사용할 보안 프로토콜 선택 (UDP, TCP, DoH, DoT, DoQ, ODoH).
  * 사전 정의된 제공자 목록(Cloudflare, Google, Quad9 등)에서 선택하거나 사용자 지정
    서버(Custom Server) 추가 인터페이스 제공.
* **속도 및 라우팅 (Routing & Speed):**
  * **캐시 관리:** DNS 캐시 활성화 여부 토글, **캐시 초기화(Purge Cache)** 버튼.
  * **Split DNS (분할 라우팅):** `.local`, `192.168.x.x` 등 로컬 네트워크나 사내망
    도메인이 외부로 유출되지 않고 OS 기본 리졸버를 타도록 우회 규칙(Bypass Rules)을
    설정하는 인터페이스.
* **플러그인 및 필터링 (Plugins & Filtering):**
  * 광고 및 트래커 차단 모듈 On/Off.
  * 유해 사이트 차단 모듈 On/Off.
  * 사용자 지정 화이트리스트(Include domains) / 블랙리스트(Exclude domains) 관리 화면.
* **일반 및 시스템 (General):**
  * OS 부팅 시 자동 시작 (Start on boot) 설정.
  * 창을 닫을 때 시스템 트레이로 최소화 (Minimize to tray on close) 옵션.
  * UI 테마 설정 (System Default, Light, Dark).

## 4. 시스템 트레이 (System Tray) 연동

앱의 메인 창이 닫혀 있더라도 백그라운드 서비스는 계속 동작하며, 트레이 아이콘을 통해 상태 확인 및
빠른 제어를 지원합니다.

* **트레이 아이콘:** 동작 상태에 따라 아이콘의 형태나 색상이 변경되어 직관적인 상태 파악을
  돕습니다.
* **컨텍스트 메뉴 (우클릭 메뉴):**
  * 현재 보호 상태 및 연결된 서버 이름 표시 (비활성화된 텍스트)
  * Start / Stop 상태 토글
  * 캐시 초기화 (Purge Cache)
  * 대시보드 열기 (Open Dashboard)
  * 앱 완전 종료 (Quit Mudfish DNS)

## 5. 권한 및 플랫폼별 통신 흐름 안내 (Permissions & OS Specifics)

* **Windows / Linux:** UI 앱은 일반 사용자 권한으로 실행됩니다. Core/Service 프로세스
  제어를 위해 관리자 권한이 필요한 이벤트(예: 최초 설치 시 시스템 데몬 등록, WFP 활성화 등)가
  발생할 경우, UI는 Service 프로세스와 IPC로 통신하며, OS가 제공하는 권한 상승
  다이얼로그(UAC, Polkit 등)를 트리거하여 진행 상황을 사용자에게 명확한 메시지로 안내해야
  합니다.
* **macOS:** App Store 샌드박스 정책에 따라 독립된 Service 데몬을 사용하지 않습니다.
  UI 앱(단일 번들) 내에서 공식 `SystemExtension` API를 직접 호출하여
  NetworkExtension을 활성화합니다. 권한 제어는 macOS가 자체적으로 사용자에게 팝업을 띄워
  대행하므로, UI는 OS API의 응답 상태에 따라 사용자에게 적절한 안내
  (예: "환경설정에서 시스템 확장 프로그램 권한을 허용해 주세요")를 제공해야 합니다.
* **모바일 (iOS / Android):** 모바일 UI는 OS의 프레임워크(NetworkExtension,
  VpnService)를 직접 호출하여 트래픽 캡처 권한을 획득하고, 내장된 Core 라이브러리를
  제어합니다.
